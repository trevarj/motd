# 07 — UI spec (Compose, Material 3, Telegram-like)

Single activity (`MainActivity`), `NavHost` with type-safe routes (`ui/nav/Routes.kt`,
contracts). ViewModels are Hilt-injected, expose a single `StateFlow<UiState>` + event
methods; composables are stateless. All user-visible strings in `strings.xml`.

## Theme (`ui/theme/`, WP6)

- `MotdTheme(themeMode, dynamicColor)`: API 31+ & dynamicColor → `dynamicLightColorScheme` /
  `dynamicDarkColorScheme`; else static schemes seeded from `#5B6EE1` (indigo).
- AMOLED = dark scheme copy with `background = Color.Black, surface = Color.Black,
  surfaceContainer* = near-black`.
- Nick colors: `nickColor(nick, isDark)` — hash nick → hue (golden-ratio spread), fixed
  saturation/lightness tuned per mode; used for sender names + avatar backgrounds.
- Avatar composable: circle, initials (first two significant chars), `nickColor` background.

## ChatList (WP6)

- TopBar: "MOTD", search icon → `SearchRoute()`, overflow → Settings.
- Connection banner under TopBar when any network not Ready ("Connecting to Libera…" /
  "Offline — reconnecting"), from `connectionStates`.
- List from `BufferRepository.observeChatList()`: pinned section on top; row = avatar,
  displayName (+ tiny network tag chip when >1 network), last message
  `sender: text` one-liner, relative timestamp, unread badge (primary), mention badge
  (tertiary, "@"), muted rows dimmed with bell-off glyph.
- FAB (+) → bottom sheet: "Join channel" (network dropdown + `#name` field →
  `connectionManager.joinChannel`), "Message user" (nick field →
  `connectionManager.ensureQueryBuffer(networkId, nick)` → navigate to `ChatRoute`).
- Empty state (no networks) → auto-navigate `OnboardingRoute`.
- `ChatListViewModel`: `data class ChatListState(val rows: List<ChatListRow>, val connection: Map<Long, IrcClientState>, val networks: List<NetworkEntity>)`.

## Chat screen (WP7) — the centerpiece

Layout: TopBar (avatar, displayName, subtitle: member count or typing indicator; tap →
`ChannelInfoRoute`) / message list / composer.

Message list:
- `LazyColumn(reverseLayout = true)` + `messages(bufferId).collectAsLazyPagingItems()`.
- **Bubbles with sender grouping**: consecutive messages from the same sender within 3 min
  form a group — sender name (nick-colored) + avatar only on the group's first bubble
  (visually bottom-most, since reversed). Own messages: right-aligned,
  `primaryContainer`; others: left, `surfaceContainerHigh`. Max width ~78%. Rounded 18dp with
  4dp inner corners within a group. NOTICE renders as a bubble with a small "notice" label;
  ACTION as italic `* nick does thing` centered-left, no bubble.
- System events (JOIN/PART/QUIT/KICK/NICK/MODE/TOPIC): centered pill chips; consecutive runs
  collapse to one pill ("3 joined · 1 left", tap to expand inline).
- Day separators (sticky feel not required); "— New messages —" divider at `readMarkerTime`.
- Reactions: chip row under the bubble (`emoji count`, own reaction filled). **Add-only in
  v1**: tapping a chip adds your reaction (`sendReact`); tapping a chip you already reacted
  with is a no-op (no removal protocol defined in v1). Reactions come from `reactions()` flow
  keyed by visible msgids.
- Message text: linkify URLs/channels/nicks (compose `LinkAnnotation`); first image URL in a
  message renders inline via Coil (`AsyncImage`, max height 280dp, tap →
  `ImageViewerRoute(url)`); first non-image URL gets `LinkPreviewCard` (async via
  `LinkPreviewRepository`, skeleton shimmer while loading, hidden on null).
- Long-press bubble → `ModalBottomSheet`: Reply, React (emoji row: 👍 ❤️ 😂 😮 😢 + "more" →
  full-ish grid of ~64 common emoji; no external emoji-picker dependency), Copy, Quote
  (prefills `> text` in composer).
- Failed messages (`failed=true`): red timestamp + retry icon.
- Reply rendering: quoted mini-bubble above text when `replyToMsgid` resolves locally
  (lookup by msgid within loaded items; else hidden).
- Scroll-to-bottom FAB with unread count when scrolled up. Typing row above composer:
  "alice is typing…" from `TypingTracker` (max 2 names + "…and others").

Composer:
- Multiline `TextField` (max ~6 lines), send `IconButton` (disabled when blank/disconnected),
  reply-preview bar with cancel when replying.
- On text change: `sendTyping(bufferId, "active")` (manager throttles); on send/cleared →
  `"done"`.
- **Nick autocomplete**: token under cursor (starting at word boundary or after `@`) matched
  prefix-wise (Isupport-normalized) against members ranked by most-recent-spoke (from loaded
  messages) then alphabetical; popup above composer, tap inserts `nick: ` at message start or
  `nick ` mid-message.
- `/` command hints: leading `/` shows popup of `/me /join /part /msg /query /nick /topic`;
  ViewModel parses: `/me x` → sendMessage("/me x") (manager translates, plans/05); `/join #c`
  → joinChannel; `/part` → partChannel; `/msg nick text` & `/query nick` →
  `ensureQueryBuffer` (+ send + navigate); `/nick n` → raw NICK via client; `/topic t` → TOPIC. Unknown `/cmd` → send
  as raw line via `clientFor(networkId).send(IrcMessage.parse(...))` after stripping `/`.
- `ChatViewModel` state: `ChatState(buffer, pagingFlow, members, typingNicks, replyTo, connState)`.
  `typingNicks` reads the `TypingTracker` contract. On resume/pause it sets/clears
  `ForegroundBufferTracker` (notification suppression, plans/05). Mark-read on resume + on
  new-message-while-at-bottom via `connectionManager.markRead(bufferId, newestVisibleTime)`
  (the single mark-read entry point). Sends replies via
  `sendMessage(bufferId, text, replyToMsgid)`. Nick normalization for autocomplete uses
  `connectionManager.clientFor(networkId)?.isupport?.normalize` (fallback: lowercase).

## Onboarding (WP8)

Internal `HorizontalPager` wizard (one route):
1. Welcome (logo, tagline, "Get started").
2. Choice cards: **"I have a soju bouncer"** / **"Connect to an IRC network"** (Libera.Chat
   preset card: host `irc.libera.chat`, port 6697, TLS — one tap prefills).
3. Server form: host/port/TLS toggle, nick, username (default = nick), realname.
4. Auth: none / SASL PLAIN (user+password) / SASL EXTERNAL (client cert alias via
   `KeyChain.choosePrivateKeyAlias`).
5. Connect test: creates the NetworkEntity (soju → BOUNCER_ROOT), `connect(networkId)`, live
   log of `IrcClientState` transitions with animated check/cross; on Ready + soju →
   `bouncerListNetworks()` shown as toggleable list (existing networks import as
   BOUNCER_CHILD rows) + "Add network" mini-form (`bouncerAddNetwork`); finish → ChatList.

## Settings (WP8)

Preference-style Compose lists. Appearance: theme mode (SYSTEM/LIGHT/DARK/AMOLED), dynamic
color toggle. Delivery: PERSISTENT_SOCKET / UNIFIED_PUSH radio (UNIFIED_PUSH disabled with
explainer when no distributor installed or no soju webpush cap), battery-optimization helper
row. Networks: list → `NetworkSettingsRoute` (edit form ≈ onboarding steps 3-4, delete with
confirm). About: version, licenses note, GitHub link.

## Search (WP8)

TopBar search field (auto-focus), filter chips (All buffers / current buffer when launched
with bufferId, sender filter via `from:nick` token parsed client-side). Results grouped by
buffer: header + hit rows (highlighted snippet, timestamp). Tap → `ChatRoute(bufferId)`;
v1 scroll-to-message: if the message is in the loaded window scroll to it, else just open the
buffer (a `CHATHISTORY AROUND` deep-jump is post-v1; leave a `// TODO(post-v1)`).

## ChannelInfo (WP8)

Header (avatar, name, topic full-text, member count), actions row (mute toggle, pin toggle,
part/leave with confirm), member list sectioned by highest prefix (`~ & @ % +` then regular,
using `client.isupport.prefixModes` order), member row tap → sheet (Message →
`ensureQueryBuffer` + navigate, Mention → prefill composer).

## ImageViewer (WP7)

Full-screen dialog route: black background, Coil image with pinch-zoom/pan
(`Modifier.graphicsLayer` + `detectTransformGestures` — hand-rolled, ~40 lines, no dependency),
share + save (MediaStore) actions, tap toggles chrome.

`ChatListRow`s; Chat renders a scripted mix (groups, system pills, reactions, reply, image,
preview card, failed msg) from a preview PagingData; no ViewModel constructed in previews.
